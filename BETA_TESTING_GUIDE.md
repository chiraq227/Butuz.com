# Butuz.com — Полное руководство по бесплатному бета-тестированию (узкий круг)

## Что было сделано перед релизом (кратко)

**Проверка и исправления (полный аудит):**
- **Безопасность**: 
  - Усилена проверка JWT_SECRET (минимум 32 символа, отказ от дефолтных плейсхолдеров).
  - Исправлен in-memory rate limiter (добавлена очистка, устранена утечка памяти).
  - Нормализованы пути загрузок (в БД теперь только basename файла, а не полный путь с диска — критично для деплоя и безопасности).
  - Улучшена обработка ошибок в продакшене (не утекают стектрейсы).
  - Добавлены базовые правила паролей (мин. 6 символов).
  - Helmet, CORS с белым списком, bcrypt, параметризованные запросы — всё на месте.
- **Шифрование (E2EE секретных чатов)**: 
  - ECDH P-256 + AES-GCM 256 (Web Crypto). Сервер хранит только ciphertext. Корректно для MVP. Ключи только в localStorage клиента.
- **Мобильные устройства**:
  - Перепроверено полностью.
  - Исправлены критические проблемы в /messages (высота экрана игнорировала нижнюю навигацию → перекрытие; двухколоночный layout был неадаптивным на телефонах → теперь на мобиле показывается либо список чатов, либо полный чат с кнопкой «назад»).
  - Увеличены touch targets (минимум ~44px) для меню постов и кнопки закрытия фото.
  - Добавлен учёт safe-area-inset (notch/Dynamic Island).
  - Глобальные стили уже содержали: font-size 16px на инпутах (нет зума iOS), touch-action: manipulation, -webkit-tap-highlight-color, overflow-x: hidden, .safe-bottom.
  - Navbar, PostCard, CreatePost, PhotoViewer, Feed, формы логина/регистрации, Casino — всё адаптивно и удобно на телефонах.
- **Оптимизация**:
  - Добавлена обработка изображений постов через sharp (resize до 1280px + WebP качество 82). Раньше только аватарки оптимизировались.
  - Сохранены чанки Vite, сжатие в проде.
- **База данных (SQLite)**:
  - Добавлены PRAGMA journal_mode=WAL, synchronous=NORMAL, temp_store=MEMORY (лучше параллелизм и скорость).
  - Добавлены индексы на горячие пути (посты, лайки, сообщения, follows).
  - Миграции безопасные (ALTER + CREATE INDEX IF NOT EXISTS).
- **Прочее**: Исправлены мелкие баги (например, отсутствовал `getDB()` в обработчике pin поста), улучшена консистентность загрузок, обновлён .env.example с чёткими инструкциями.

Код готов к бета-релизу. Рекомендуется **перед раздачей ссылок** выполнить `npm run seed` (чистая база + удаление старых загрузок).

## Самый лучший способ запустить БЕСПЛАТНО для узкого круга (рекомендую)

**Вариант A — Локальная машина + бесплатный туннель (Cloudflare / ngrok) — ИДЕАЛЬНО для бета-теста**

Преимущества:
- Полностью бесплатно.
- Данные (SQLite + uploads + аватарки) сохраняются на твоём диске (никаких потерь при рестартах).
- Полный функционал (включая большие файлы, казино, E2EE).
- Легко обновлять код и сразу видеть изменения у тестеров.
- HTTPS из коробки.
- Идеально для 5–30 человек.

Минусы: Пока твой ПК/ноутбук включён и процесс запущен — сайт доступен. (Можно оставить на ночь/выходные.)

### Пошаговая инструкция (Windows + PowerShell)

1. **Подготовка кода (сделай один раз)**

   Открой PowerShell в папке проекта `C:\Users\6\Desktop\Butuz.com-main`

   ```powershell
   # 1. Убедись, что зависимости стоят
   npm install
   cd server; npm install; cd ..
   cd client; npm install; cd ..

   # 2. Сгенерируй СИЛЬНЫЙ JWT_SECRET (скопируй вывод)
   node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
   ```

   Открой `server/.env` (блокнотом):

   ```powershell
   notepad server\.env
   ```

   Замени строку:
   ```
   JWT_SECRET=твой_сгенерированный_очень_длинный_хэш_сюда
   ```

   (Остальное можно оставить по умолчанию для бета. CLIENT_URL можно не трогать при использовании туннеля.)

   Сохрани файл.

2. **Очисти базу для чистого бета-теста (ВАЖНО!)**

   ```powershell
   npm run seed
   ```

   Это удалит все старые посты, пользователей, загрузки. Потом первые регистрации будут "чистыми".

3. **Запусти сайт в режиме разработки (или прод)**

   ```powershell
   npm run dev
   ```

   Должны запуститься два процесса:
   - Backend на :5001
   - Frontend (Vite) на :5173

   **Важно**: Оставь это окно открытым. Не закрывай терминал.

   (Альтернатива для "продакшен-подобного" режима: сначала `npm run build` (в корне или в server), потом `cd server; $env:NODE_ENV="production"; npm run start:prod` — тогда будет использоваться собранный клиент.)

4. **Выдай публичную ссылку друзьям (бесплатный туннель)**

   **Рекомендуемый вариант — Cloudflare Tunnel (cloudflared)** — не требует аккаунта для простого использования.

   Скачай cloudflared для Windows: https://github.com/cloudflare/cloudflared/releases (файл cloudflared-windows-amd64.exe, переименуй в cloudflared.exe и положи в PATH или в папку проекта).

   Затем в **новом** окне PowerShell (пока dev-сервер работает):

   ```powershell
   # Из папки проекта
   .\cloudflared.exe tunnel --url http://localhost:5173
   ```

   В выводе найди строку вида:
   ```
   https://random-words-1234.trycloudflare.com
   ```

   Это и есть публичная HTTPS-ссылка. Отправь её тестеровщикам.

   Они открывают ссылку → видят сайт полностью (Vite dev проксирует /api и /uploads на твой бэкенд).

   **Альтернатива (ngrok)**:
   - Зарегистрируйся на https://ngrok.com (бесплатно)
   - Скачай ngrok, получи токен
   - `ngrok http 5173`

5. **Что сказать тестерам**

   - Зарегистрируйся под уникальным именем (например, test-ivan-01).
   - Можно загружать фото/видео в посты и сообщения (до ~8-15 МБ).
   - Секретные чаты: оба должны хотя бы раз зайти во вкладку «Секретные чаты», чтобы обменяться ключами.
   - Казино: стартовый баланс большой для тестов.
   - Сообщай о багах (скрины, шаги воспроизведения).

6. **Как остановить / перезапустить**

   В терминале с `npm run dev` — Ctrl+C.
   Чтобы перезапустить после правок кода — просто снова `npm run dev`.

   После обновления кода друзья просто перезагружают страницу (или ты им скажешь).

## Другие бесплатные варианты (с оговорками)

**Вариант B — Render.com (Free tier)**

- 750 часов/месяц, засыпает после 15 мин бездействия.
- **Проблема**: на бесплатном тарифе диск часто эфемерный — после деплоя или долгого сна SQLite + uploads могут пропасть. 
- Решение: перед деплоем всегда делай `npm run seed` или используй внешнюю БД (сложно).
- Подходит, если тестеры активны часто.

**Вариант C — Railway / Fly.io / Koyeb** (хобби/бесплатные кредиты)

- Иногда дают persistent volume даже на бесплатных планах.
- Нужно проверить текущие лимиты на момент запуска.

**Вариант D — Oracle Cloud Always Free (самый "вечный" бесплатный)**

- 2 ARM VM + до 200 ГБ блочного хранилища бесплатно навсегда.
- Требует карту при регистрации (не спишут, если не превысить).
- Сложнее настройка (Linux, PM2 или systemd, nginx/caddy, открытие портов или Cloudflare).
- Если нужен "всегда доступен" без твоего ПК — лучший долгосрочный бесплатный вариант.

## Безопасность для бета

- Дай ссылку только доверенным людям.
- Попроси сильные пароли при регистрации.
- Админ-панель казино работает только для username'ов из `ADMIN_USERNAMES` в .env (по умолчанию "butuz").
- В реальном релизе добавь: rate limiting по пользователю, CAPTCHA, email verification, revokable токены и т.д.

Удачи с бета-тестом! Если найдутся баги — пиши, быстро починим.

После теста можно будет деплоить на нормальный хостинг с persistent volume + доменом.
