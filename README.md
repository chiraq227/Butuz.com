# Butuz.com 🐾

**Личная социальная сеть** — минималистичный MVP на React + Node.js + SQLite.

Чистый современный дизайн в стиле актуальных соцсетей с синим/фиолетовым акцентом.

## Возможности первого макета

- Регистрация и вход (JWT)
- Главная лента (Feed) с постами
- Создание постов (текст + изображение до 5 МБ)
- Лайки на посты (toggle)
- Удаление своих постов
- Страница профиля пользователя (аватар, имя, био, список постов)
- Адаптивный дизайн (мобильный + десктоп)
- Все данные хранятся в локальной SQLite базе

## Быстрый старт (пошагово)

> **Рекомендуемый способ (Windows / PowerShell / один терминал)**

### 1. Установка всего одним командами (из корня проекта)

```powershell
# Из папки Butuz.com-main
npm install                 # установит concurrently + deps сервера и клиента
npm run install:all         # на всякий случай (если нужно переустановить отдельно)
```

### 2. Запуск (одной командой)

```powershell
npm run dev
```

Это запустит **одновременно** и сервер, и клиент (с цветными префиксами в консоли).  
Чтобы остановить — `Ctrl + C`.

Альтернативные команды из корня:
- `npm run dev:server` — только бэкенд
- `npm run dev:client` — только фронтенд
- `npm run seed` — очистка базы (см. ниже)

### 3. Настройка переменных окружения

`.env` уже создан в `server/.env` (я скопировал из `.env.example` для удобства).

При необходимости отредактируй:
```powershell
cd server
notepad .env
```

Можно оставить значения по умолчанию для локальной разработки.

### 4. Полная очистка базы данных (рекомендуется при первом старте)

**Важно:** 

1. **Останови все процессы** (`npm run dev`) перед запуском очистки.
2. Выполни:

```powershell
npm run seed
# или из папки server:
# npm run seed
```

Эта команда полностью очищает:
- Базу `butuz.db` + WAL/SHM файлы
- Папку `server/uploads/` (аватарки, изображения постов)
- Все таблицы (пользователи, посты, лайки, сообщения, казино-данные и т.д.)

После этого заново запусти `npm run dev` и зарегистрируй первого пользователя.

### 5. Откройте в браузере

Перейдите по адресу: **http://localhost:5173**

### Альтернатива: два отдельных терминала (классический способ)

**Терминал 1 — Backend:**
```powershell
cd server
npm install
npm run dev
```

**Терминал 2 — Frontend:**
```powershell
cd client
npm install
npm run dev
```

### 6. Дополнительно

- `npm run seed` — полная очистка (останови сервер сначала!)
- После изменений в клиенте Vite сам перезагрузит страницу (HMR).
- Сервер использует `node --watch` (автоперезапуск при изменении файлов).

## Основные эндпоинты API

- `POST /api/auth/register` — регистрация
- `POST /api/auth/login` — вход
- `GET  /api/posts` — лента
- `POST /api/posts` — создать пост (multipart, поле `image` опционально)
- `POST /api/posts/:id/like` — поставить/убрать лайк
- `GET  /api/users/:username` — профиль + посты пользователя
- `GET  /api/users/me` — текущий пользователь (требует Authorization)

**Сообщения (обычные + секретные)**
- Обычные: `POST /api/messages`, `GET /api/messages/:userId`, `GET /api/messages/conversations`
- Секретные (E2EE): `POST /api/messages/secret`, `GET /api/messages/secret/:userId`, `GET /api/messages/secret/conversations`
- Ключи (для секретных): `POST /api/messages/keys`, `GET /api/messages/keys/:userId`

## Технологии

**Frontend:**
- React 18 + Vite + TypeScript
- Tailwind CSS
- React Router v6
- Lucide icons

**Backend:**
- Node.js + Express
- SQLite (через `sqlite` + `sqlite3`)
- JWT (jsonwebtoken) + bcryptjs
- Multer для загрузки изображений

## Что можно улучшить в следующих итерациях

- Комментарии к постам
- Редактирование аватара и смена пароля
- Пагинация ленты
- Реальные уведомления (WebSocket)

**Уже реализовано в этом обновлении:**
- Друзья / подписки (страница + кнопки в профиле)
- Закладки (сохранение постов + отдельная страница)

- Настройки профиля + 10 цветовых тем оформления (живой переключатель + сохранение)
- **Сообщения** — новая вкладка с двумя режимами:
  - **Обычные чаты**: мгновенный обмен (как посты/комментарии). Работает сразу между любыми пользователями, не нужно быть онлайн одновременно.
  - **Секретные чаты**: отдельный режим с полным сквозным шифрованием (ECDH P-256 + AES-GCM). Сервер видит только шифртекст. Чётко помечено, требует активации.
